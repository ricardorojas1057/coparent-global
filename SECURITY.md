# Seguridad

## Reportar una vulnerabilidad

No publiques datos personales, tokens, contrasenas ni pruebas con informacion familiar en un issue
publico. Reporta la vulnerabilidad de forma privada al propietario del repositorio.

Inclui una descripcion reproducible, el impacto posible y una propuesta de mitigacion si la tenes.

## Datos sensibles

Este repositorio no debe contener:

- Archivos `.env`.
- `google-services.json` o claves de servicio Firebase.
- Keystores o claves de firma Android.
- Credenciales de Vercel, Neon, Expo, Google o Meta.
- Mensajes, comprobantes o datos reales de familias.

Las notificaciones y reportes de errores deben evitar contenido familiar sensible por defecto.
